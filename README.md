보안 필터링의 오탐률을 줄이고 차단 시 투명성을 높이기 위해 engine.py, app.py, templates/index.html 세 개의 파일을 수정했습니다. 자세한 변경 사항은 다음과 같습니다.

1. engine.py (보안 필터링 엔진 개선)
DOM Sanitizer 오탐 방지:
기존에는 <script> 및 <style> 태그 내부의 텍스트까지 추출하여 검사 대상에 포함시켰기 때문에 안전한 소스코드나 라이브러리(예: GitHub 페이지)도 악성 코드로 오탐되는 문제가 있었습니다.
이 스크립트/스타일 태그들은 어차피 LLM에 전달되지 않고 파기되므로, 검사 텍스트 수집 대상에서 제외하고 즉시 완전 삭제(decompose())하여 오탐의 주원인을 해결했습니다.
검사 규칙의 이원화 (외부 데이터 vs 사용자 쿼리):
사용자가 직접 입력하는 쿼리(User Query)에서는 역할 변경이나 모드 지시(act as, you are now, 역할변경해줘 등)가 정상적인 기능 작동 과정일 수 있습니다. 따라서 이 패턴들은 외부 데이터(HTML 본문) 검사 시에만 차단되도록 분류를 나눴습니다.
반면, jailbreak, DAN, 시스템 프롬프트 출력, 이전 지시 무시와 같은 명백한 악성 공격 구문은 사용자 쿼리와 외부 데이터 모두에서 엄격하게 탐지 및 차단됩니다.
상세 차단 정보(blocked_info) 추출:
공격이 감지되었을 때 단순 차단 여부(blocked: True)만 반환하는 것이 아니라, **탐지 레이어명, 매칭된 패턴 정규식, 탐지된 영역(외부 데이터/숨겨진 요소/사용자 쿼리/출력 가드레일), 그리고 매칭된 텍스트 주변의 60자 맥락 스니펫(snippet)**을 하나의 객체로 수집하여 반환하도록 변경했습니다.
2. app.py (백엔드 API 컨트롤러)
/api/process API가 engine.process_request()로부터 상세 차단 정보(blocked_info) 객체를 추가로 받아오도록 수정했습니다.
검증 결과 및 차단 여부를 클라이언트에 전달할 때 blocked_info도 JSON 응답에 포함시켜 전달하도록 업데이트했습니다.
3. templates/index.html (프론트엔드 UI 시각화)
차단 시 기존의 불투명한 에러 메시지 대신 상세 분석 리포트를 제공합니다.
차단된 영역 (예: 가시 외부 데이터, 숨겨진 텍스트 등), 탐지 레이어, 탐지된 패턴을 명확히 명시합니다.
특히 탐지된 본문 스니펫에서 정규식 매칭을 이용해 어떤 단어/문구 때문에 차단되었는지 빨간색 배경과 밑줄로 하이라이트 처리하여 보여주도록 개선했습니다.
